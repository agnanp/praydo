# Product Guidelines: Praydo

## 1. Prose Style

- **Concise & Helpful:** All user-facing text, including labels, descriptions, and notifications, should be clear, brief, and action-oriented. Avoid unnecessary jargon or long explanations.

## 2. Branding & Tone

- **Modern & Respectful:** The application should maintain a trustworthy and respectful tone, appropriate for its purpose as a prayer time utility.
- **Efficient & Minimalist:** The branding should convey a personality focused on speed and ease of use, ensuring that the interface is not cluttered.

## 3. User Experience (UX) Principles

- **Simplicity First:** Prioritize a user journey that is straightforward and easy to navigate. The interface should be intuitive, requiring minimal cognitive effort from the user.

## 4. Visual Style

- **Modern (Skeleton UI):** Leverage the Skeleton UI library combined with Tailwind CSS to create a modern, web-native desktop interface. This includes consistent use of components, spacing, and typography that align with these frameworks.

## 5. Accessibility & Inclusivity

- While not the primary focus, the application should aim for standard accessibility practices to ensure usability for a broad audience.
